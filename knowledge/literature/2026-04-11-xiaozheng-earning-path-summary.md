## Core Idea

赚钱本质上是概率游戏，个体创业者应当用假设驱动的方式拆解业务、识别核心假设、低成本验证，从而系统化地提高赚钱的成功概率。

## Key Points

1. **赚钱是概率游戏**：每个人的资源、背景、社会关系都影响赚钱概率，大部分人搞钱的动作是"歪的"，缺乏对搞钱本身的完整思考。X型搞钱人直接执行不思考假设，Y型搞钱人用假设驱动推进项目。
2. **假设与核心假设**："假设"是项目成立所依赖的前提条件；"核心假设"是所有假设中风险最高、对赚钱结果影响最重要的假设。拆解的关键原则是"拆不拆差别很大，拆得好不好差别没那么大"。
3. **五步法框架**：列出所有假设 → 做减法找到核心假设 → 优先验证前置假设 → 尽量先验证单个假设 → 哪个风险高优先验证哪个。
4. **创新是被逼无奈的手段**：在赚钱的世界里，但凡能通过抄去解决问题的，一律不建议创新。一切思考都要回归到研究"需求"。
5. **低成本验证策略**：三条验证路径——靠常识、靠调研、靠试验。成本排序为：常识 < 调研 < 验证 < 全量开发。
6. **快钱不可持续**：需要构建长期壁垒，而非追求短期暴利。

## Evidence

- 2005年做打车软件的假设分析案例：说明未经验证的假设会导致巨大资源浪费。
- 企业信用修复项目的假设验证案例：展示五步法在实际业务中的应用。
- 懒羊羊占卜案例：通过五步法筛选出3条核心假设，一个月赚了6000元，验证了假设驱动方法的有效性。
- 投屏电视盒子案例：1年研发 + 200万投入，最终只出几单，是未验证假设的典型反面教材。
- 小挣青年创立过程：核心假设为"获客是否能持续稳定获取流量"，通过验证该假设推动业务落地。

## Open Questions

- 核心假设的识别是否有更系统化的方法，还是完全依赖经验判断？
- 低成本验证中的"常识"边界在哪里——如何判断一个假设是否属于常识？
- 五步法在不同行业、不同规模项目中的适用性差异如何？
- "抄"与"创新"的边界如何界定——什么情况下抄也会带来风险？
