# 巡检提示词

你正在执行"巡检"步骤。

## 目标

对现有知识系统进行结构性审计，找出问题并生成修复建议报告。

## 要求

1. 你现在的职责是审计，不是直接重写整个知识库。
2. 优先发现结构问题：冲突、重复、孤岛、断链、过时、缺来源。
3. 每个问题都要给出证据。
4. 每个问题都要给出建议动作。
5. 不要自动修复，除非被明确要求。
6. 输出必须符合巡检报告 schema（参见 `schemas/inspection-schema.md`）。

## 巡检顺序

1. 读取 `knowledge/index/index.md`。
2. 扫描指定目录。
3. 找出问题。
4. 给出优先级（High / Medium / Low）。
5. 输出报告。

## 检查项

1. 是否存在定义冲突（同一概念在不同文件中定义不一致）。
2. 是否存在重复概念（应该合并的条目）。
3. 是否存在没有被引用的孤岛文件。
4. 是否存在缺少来源支持的结论。
5. 是否存在长期未更新但仍被高频引用的内容。
6. 双向链接 `[[]]` 是否正确。
7. frontmatter 字段是否完整一致。

## Schema 参考

参见 `schemas/inspection-schema.md`

## 关键原则

> 巡检先出报告，不要默认自动改。
> 审计员的职责是暴露问题，而不是悄悄修改。
