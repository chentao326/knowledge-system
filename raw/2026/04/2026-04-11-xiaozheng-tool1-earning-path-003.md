**理解：搞钱的背后到底是在搞什么？**

不知道有没有人认真的去思考一个问题，我们在公司管业务也好，或者自己去做项目也好，我们究竟在做什么？或者究竟在为什么而做？辛苦半天，赚不到钱，**回顾过去一年自己不管是经验还是金币都没有特大的增长。**背后控制这套运算逻辑的究竟是什么？

提到赚钱，一定避不开的就是具体**"搞钱的动作"**。

可大部分人并没有意识到自己搞钱的动作是有欠缺/变形的。我们都对**"搞钱的动作"**这一事项缺少了**完整思考。**

那是否还有其他思考路径呢？

我访问了一些优秀的伙伴们，包括如今的我自己，我发现我们都有一个很好的思维习惯

**我把它叫做：赚钱可以用假设推动。**

我们在沟通讨论赚钱相关话题的时候常常喜欢说一句话：**"你这个业务/渠道/产品/项目的『核心假设』是什么?"**，

我把习惯这种思考方法的小伙伴者，分为**Y型搞钱人**

什么叫假设？就是**不要看项目表面的现象：表面的数据、表面的形态、表面的用户评价；**

而是一切都围绕**前提假设**来进行项目的推进和落地。

  ----------------------------------------------------------------------------------- -------------------------------------------------------------------------
     
  ----------------------------------------------------------------------------------- -------------------------------------------------------------------------

【重点】这里所谓的假设，就是你想通过这件事赚到钱，他所依赖的**『前提条件』**

换句话来说，如果这些假设全部验证成立，**你的项目一定能赚到钱。**

而如果有一条不成立，那你的项目就很难走下去，你需要想办法止损转型，或者放弃。

在假设之上，又有一个词语，叫做**『核心假设』**

**『必学：工具』核心假设，是所有假设里面，风险最高，对你赚钱结果影响最重要的假设。**

说句实话，想到一个idea或者看到一个项目**这件事本身一点都不值钱。**

太阳底下没有新鲜事，全中国至少有成百上千个人和你有过同样的想法，这个并不值钱。

真正值钱的，是这个idea背后，**一整套的【核心假设】。**

**假设在2005年，那时候还没有智能手机，你想做一个网页版本的打车软件。**

------------

**那你可能会设立的假设或许是以下这些：**

用户路边招手打车不方便，存在痛点（市场需求）

大量私家车司机存在空跑，愿意网上接点活（可能也存在）

**新互联网服务打车整体体验，远远好于路边招手模式（司乘双方体验均是）**

杜绝蓝牌车和黑的士私自加价，统一收费模式

WEB版打车产品，可以通过线下地推或者百度SEO，快速获客增长

**掌握"核心假设"方法：**

其实很多项目在初期并不是奔着要赚具体多少钱来的，更多是想**先测通**然后在通过**加杠杆或者减少不必要动作**来提升自己的赚钱效率，当你会使用假设思维之后，你可以大幅度降低项目探索的成本，可以减少在项目进度中的负担，直至这个项目能赚到钱或者帮你实现自己目的。

好，我们下面开始今天的实操，在实操过程中，**我们要掌握这个核心假设方法。**

**【重点】**我们要用到一个核心流程：**先加，后减，快速验证**

**三个环节分别有三个套路**

**第一步：【先加】**像我们一开始就提到的，你需要先把你做的项目或者业务，拆成一组假设，用几句话，或者一页纸，把整个要素都提炼出来；我们学三个方法

**第二步：【后减】**你得有能力从中挑出来重点，不能漫无目的一把抓，得把注意力放在核心假设上；我们学三条原则

**第二步：【快速验证】**面对挑出来的核心假设，我们要学会如何快速验证，如何用10%的投入，完成100%的工作；我们学三个策略

**第一步：先加**

如何分解一个项目，把里面的假设一条条的都提炼出来呢？

**很简单，要明白假设是可以分类的**

在拆解过程当中，我们会发现始终有一类假设，围绕着**【价值】**进行设立；

|                                                |

就和樊登老师在***《低风险创业》***这本书里提到的一个很有趣的点一样，**最开始樊登也不清楚，提供一个帮大家讲书和拆书的服务，真的有价值吗？**

樊登老师自己说的，樊登读书会的第一个版本，其实就是一个PPT，然后看大家愿不愿意付费。

事实是，即使是这么粗糙的方案，**很多人依然愿意花300块钱来买这个PPT**，说明这个事情，**有价值。**正因为证明了有价值，所以樊登老师才一步步把这个产品做了起来。

**所以我们学习到一个新的词汇：价值假设**

伴随着业务或者项目有价值的话，新的问题就来了：我的这个业务怎么样**快速地复制、快速地扩张、快速地传播**开来，让大家都来使用呢？

这个时候，我们就开始进入到了项目的**【增长】**环节。

**没错，第二个新词就是：增长假设**

很多人在聊天的时候经常会聊自己的营销策略，或者裂变路径，但他们经常会**忽略自己的价值假设是否有验证过**。如果没有验证价值假设，其实后面所有关于增长的假设，意义都不大。

因为增长的前提条件是，你的产品需要有真正的价值，**不然过早的去复制，你扩张的越快，你亏损的越多，反而创业或者项目失败也就越快。**

  ------------------------------------------------------------------------------------ ------------------------------------------------------------------------------------

**原则一：优先验证前置假设**

优先验证前面的**前置假设**

我们没有太多创业经验或者搞钱经验的人，下意识的会提更多关于**【运营增长】的想法和idea**，我们都习惯去思考增长问题，忽略了在增长前面还有**三个更大的前置思考，三个价值相关的假设。**

**还有的人，构思商业模式时总是在考虑【创新】，其实很多时候，创新是被逼无奈的手段，而不应该是目的；**通常情况下，创新一个需求或者项目最大的风险往往就是**没有需求。**

所以一切的思考，都要回归到研究**『需求』**开始。

+--------------------------------------------------------------------------+
| **赠与各位一个提醒：**                                                   |
|                                                                          |
| 在赚钱的世界里面，但凡能通过抄去解决问题的，一律不建议创新。             |
|                                                                          |
|   ---------------------------------------------------------------------- |
|   创新只是为了达成自己目的的手段，而且是被逼无奈的手段，而不应该是目的   |
|   ---------------------------------------------------------------------- |
+--------------------------------------------------------------------------+

如果你真的按照一开始的增长思维去做项目或者搞钱，你就很容易陷入一直思考获客增长的问题而忽略了它的前置假设；会导致在创业的过程中，经常性的出现用户画像不准，解决方案要调整，等等。

  ------------------------------------------------------------------------- ------------------------------------------------------------------------------------
     
  ------------------------------------------------------------------------- ------------------------------------------------------------------------------------

更有甚者，因为需求定位出了问题，导致业务模式几乎被打回最初重头再来，**轻则损耗时间，重则劳民伤财。浪费很多不必要的成本。**

**原则二：尽量先验证单个假设**

假如现在你在创业，你想做一个产品**（比如紫苏牛蛙预制菜）**，但是**市场需求不确定性非常高。你不知道这个需求存不存在。**

**如果是你，你会怎么做？**

+-----------------------------------------------------------------------------------+
| A：三个月先把产品做出来，同时验证**需求**和**解决方案**，开发过程大概要**3个月**  |
|                                                                                   |
| B：**1-3周搞个PPT（或者发个问卷）**，只验证**需求**，解决方案等需求摸清楚了在说吧 |
+-----------------------------------------------------------------------------------+

------------------

事实上，大部分人都知道B好，**但现实往往选A的人更多**

  ------------------------------------------------------------------------------------ ------------------------------------------------------------------------------------- ------------------------------------------------------------------------------------
        
  ------------------------------------------------------------------------------------ ------------------------------------------------------------------------------------- ------------------------------------------------------------------------------------

举个很真实的案例，有个客户制作了一个**投屏电视盒子**；

**研发时长1年加上研发资金约200多万；**

通过插到电视可以实现手机投放自己的视频在电视

但最终结果，只出了几单......

**我们并不是说A类型绝对不好，而是大部分时间，A的试错成本，高于B；**

**比如摆摊、闲鱼卖货等等。试错成本较低的情况下，可以尝试用A方法，取决于你验证假设的投入成本。**

**原则三：哪个风险高，优先验证哪个**

如果你的方案里面，有1-2个非常高风险的假设选项，一定优先验证。

**一句话解释：**如果你的方案里面，有一到两个问题你是觉得**考虑不清楚（说不出来）**的，那么很有可能这个考虑不清楚的内容就是**高风险的假设。**

**无法回答或者考虑不清楚**，就证明你的模式和项目中存在**巨大的风险**；而这个风险验证，拖得越晚，损失越大。同样的理论，**在你辨别一个项目是否是割韭菜的时候，一样有用。**

**第二条策略：靠调研**

有一些答案可能搜不到，或者你获取不到，那么你就可以尝试通过调研一些竞争对手或者市场潜在的目标用户，来拿到一些关键的数据结果；这也是为什么海外问卷调研能够有一定市场的原因。有价值的调研可以帮你节省几倍甚至十几倍的时间。
